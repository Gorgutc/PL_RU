# PL_RU Codex Bootstrap

Use this when starting a new Codex session in this repository.

1. Read `AGENTS.md`.
2. Check branch and status with `git status --short --branch`.
3. Identify whether the task is narrow or broad.
4. For narrow frontend work, use `pl-ru-frontend-rules`.
5. For broad audits or instruction/tooling changes, use
   `docs/agent/orchestration.md` and the audit roles listed there.
6. Treat `.codex/` as a convenience mirror, not the source of truth.
7. Before delivery, run the appropriate quality command and report failures.
